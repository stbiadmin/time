# Data Dictionary

## RMA Shipping Data (`data/raw/rma_shipping_data.csv`)

Synthetic dataset simulating 2 years of RMA (Return Merchandise Authorization) shipping records.

### Schema

| Column | Type | Description |
|--------|------|-------------|
| `date` | datetime | Date of RMA request |
| `region` | string | Geographic region (NA, EMEA, APAC, LATAM, ANZ) |
| `sku_category` | string | Part category (CPU, GPU, RAM, etc.) |
| `shipping_weight_kg` | float | Individual shipment weight in kg |
| `request_urgency` | int | Urgency level (1=standard, 2=expedited, 3=critical) |
| `shipping_method` | string | Shipping method (ground, express, air) |
| `avg_repair_cycle_days` | float | Average repair cycle time for this SKU |
| `failure_rate_pct` | float | Current failure rate for this SKU |
| `day_of_week` | int | Day of week (0=Monday, 6=Sunday) |
| `month` | int | Month of year (1-12) |
| `is_quarter_end` | int | Whether date is in last 3 days of quarter |

### Embedded Patterns

1. **Weekly Seasonality**: Lower volume on weekends
2. **Monthly Patterns**: Spike at end of month/quarter
3. **Regional Trends**: APAC growing 8%/year, NA stable
4. **SKU Characteristics**: Different base weights and volatility
5. **Urgency Correlation**: Higher urgency → faster shipping method

### Statistics

- **Records**: ~50,000
- **Date Range**: 2 years
- **Regions**: 5
- **SKU Categories**: 15
- **Weight Range**: 0.1 - 50 kg

## Network Events Data (`data/raw/network_events.csv`)

Synthetic dataset simulating 30 days of network log events with 6 distinct behavioral clusters.

### Schema

| Column | Type | Description |
|--------|------|-------------|
| `timestamp` | datetime | Event timestamp |
| `source_ip` | string | Source IP address |
| `dest_ip` | string | Destination IP address |
| `port` | int | Network port (1-65535) |
| `duration_ms` | float | Connection duration in milliseconds |
| `bytes_transferred` | int | Total bytes transferred |
| `protocol` | string | Network protocol (TCP, UDP, ICMP) |
| `severity` | string | Event severity (info, warning, error, critical) |
| `log_message` | string | Free-text log message |
| `hour_of_day` | int | Hour of event (0-23) |
| `day_of_week` | int | Day of week (0=Monday, 6=Sunday) |
| `true_cluster` | string | Ground truth cluster label |

### Embedded Clusters

| Cluster | Proportion | Characteristics |
|---------|------------|-----------------|
| `normal_web` | 35% | HTTP/HTTPS, short duration, moderate bytes |
| `normal_db` | 20% | Database ports, very short, internal only |
| `suspicious_scan` | 10% | Port scanning, tiny packets, external source |
| `auth_failure` | 12% | SSH/LDAP ports, error severity, failed logins |
| `data_exfil` | 8% | Long duration, high bytes, external destination |
| `maintenance` | 15% | Off-hours, internal, info severity |

### Log Message Templates

Each cluster uses distinctive vocabulary:
- **Web traffic**: "GET request", "HTTPS", "API", "session"
- **Database**: "SELECT query", "transaction", "connection pool"
- **Suspicious**: "port scan", "enumeration", "blocked"
- **Auth failure**: "authentication failed", "password retry", "lockout"
- **Exfiltration**: "large transfer", "bulk download", "external"
- **Maintenance**: "backup completed", "log rotation", "health check"

### Statistics

- **Records**: ~30,000
- **Time Window**: 30 days
- **Unique Source IPs**: ~1,000
- **Unique Dest IPs**: ~500
- **Protocol Distribution**: TCP 85%, UDP 10%, ICMP 5%

## Processed Data (`data/processed/`)

After preprocessing, data is stored in formats optimized for model training:

- **RMA**: Time series sequences with windowing
- **Network**: Combined numerical + LSA text features

These files are generated by the preprocessing modules during training.
